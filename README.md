# 理念
つくる、より良くつくる。

## 理想
1. つくるとは思考の実装
2. 思考は言葉
3. 余白は思考を補完する
4. 伝えろ
5. 良さは体を超える


## 方針
1. 課題を明らかにする
2. なぜその課題を解決するかを論じる
3. 課題解決の手法として複数の選択肢を挙げ、違いを論じる
4. 手法を選択した理由を論じる
5. 用いた手法によって課題が解決されたかを論じる
6. 第三者が出力したコードを扱うときは十分に理解して使う
7. 「十分に理解」とは、自分の言葉で説明ができるということである
8. 方針はアップデートされる
